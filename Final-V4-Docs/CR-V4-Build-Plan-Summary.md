## MOST CRITICAL POINTS (DON'T FORGET)

### 1. This is TDD (Tests First)
Every piece of code starts with a test. You write the test that fails, then make it pass.

```python
# WRONG: Write code, then test
def select_next_question():
    # ... implementation
    pass

# RIGHT: Test first, then code
def test_selector_chooses_right_difficulty():
    # This test fails right now
    assert something == expected
    
# Then implement to make test pass
def select_next_question():
    # ... implementation
```
